## 1. Claude AI 개요

### 1.1 Claude란?

**Anthropic**에서 개발한 생성형 AI 챗봇 서비스

| 항목 | 내용 |
|------|------|
| 개발사 | Anthropic |
| 설립 | 2021년 |
| 설립자 | 다리오 아모데이(Dario Amodei) — OpenAI 전 부사장 |
| 이름 유래 | Claude Shannon (정보 이론의 아버지) |
| 핵심 철학 | **"안전하고 정확한 AI"** |

**설립 배경**
- OpenAI 연구팀 출신 연구자들이 AI의 잠재적 위험성을 인식하고 독립 창업
- AI가 인간에게 신뢰할 수 있는 도구가 되도록 윤리적·안전한 방향을 추구

---

### 1.2 Claude AI 주요 특징

| 특징 | 설명 |
|------|------|
| **윤리적 AI 설계** | 안전하고 책임 있는 방식으로 응답 |
| **Artifacts** | 대화 중 코드, 문서, 웹사이트를 실시간으로 생성 및 미리보기 |
| **200K 토큰 컨텍스트** | 약 15만 단어(소설 한 권 분량)를 한 번에 처리 |
| **환각(Hallucination) 최소화** | 모르는 것은 모른다고 솔직하게 답변 |
| **Extended Thinking** | 복잡한 문제를 단계별로 사고하는 심층 추론 기능 |

---

### 1.3 Claude 모델 비교

Claude는 용도에 따라 3가지 모델로 나뉜다. 모델 이름은 문학 형식에서 영감을 받았다.

| 모델 | 문학적 의미 | 특징 | 적합한 작업 | 추천 사용자 |
|------|-----------|------|------------|------------|
| **Opus 4.6** | 대작 (긴 시) | 최고 성능, 복잡한 추론 | 연구·논문·고급 분석 | 연구자·전문가 |
| **Sonnet 4.6** | 정형시 (14행) | 성능-속도 균형 | 교육·실무·개발 | 일반 사용자·개발자 |
| **Haiku 4.5** | 짧은 시 (3행) | 빠른 응답, 저비용 | 챗봇·FAQ·실시간 처리 | 서비스 운영자 |

**선택 기준 요약**
- 복잡한 문제, 정밀성 필요 → **Opus**
- 일반적인 사용, 균형 → **Sonnet** (기본값)
- 빠른 응답, 간단한 작업 → **Haiku**

> **확장 사고(Extended Thinking)**: 추론을 출력과 분리하여 응답 전에 내부에서 충분히 생각할 공간을 주는 기능. 복잡한 문제에 효과적.

## 2. claude.ai 기본 활용

### 2.1 기본 인터페이스

```
[왼쪽 사이드바]          [가운데 대화창]
- 새 채팅               - 텍스트 입력창
- 프로젝트              - (+) 파일 업로드
- 아티팩트              - 모델 선택
- 로그인 계정
```

**파일 추가 메뉴 (+)**
- 파일 또는 사진 추가
- 스크린샷 캡처하기
- 프로젝트에 추가
- Google Drive에서 추가
- GitHub에서 추가
- 스킬 / 커넥터 설정
- 웹 검색 ON/OFF

### 2.2 설정 (개인 맞춤화)

설정 → 일반 → **개인맞춤설정**에서 Claude의 응답 스타일을 사전에 지정할 수 있다.

**설정 예시 (교수 사례)**
```
신원/역할 — 인공지능소프트웨어학과 교수
전문 분야 — ML/DL, NLP, 컴퓨터 비전, 프로그래밍 언어
주요 사용 목적 — 강의 자료 준비, 논문 작성·검토, 코드 디버깅, 학생 질문 답변
응답 스타일 — 학술적·정확한 표현 우선, 전문 지식 전제
```

> 이렇게 설정해두면 매번 배경 설명을 입력하지 않아도 된다.

### 2.3 프로젝트 (Project)

관련된 대화들을 **하나의 작업 공간**으로 묶어서 관리하는 기능

> 무료 사용자는 최대 **5개** 프로젝트 생성 가능

**핵심 구성 요소**

| 요소 | 설명 | 예시 |
|------|------|------|
| **지속적 메모리** | 프로젝트 내 모든 대화에서 공유되는 System Prompt 설정 | "이 프로젝트는 NLP 강의 자료 개발용, 수강생은 학부 3학년" |
| **파일 업로드** | PDF, 코드, 문서를 업로드하면 모든 대화에서 참조 가능 | 논문, 강의계획서 한 번 업로드 후 반복 활용 |
| **대화 분류** | 주제별로 대화를 프로젝트로 구분 | 강의 준비 / 연구 논문 / 코드 디버깅 |

**프로젝트 지침 사용 시기**
- 프로젝트별 맥락을 제공할 때
- 특정 워크플로에 대한 지침을 설정할 때
- Claude가 맡아야 할 역할이나 관점을 정의할 때

### 2.4 아티팩트 (Artifacts)

Claude가 **대화창 옆 별도 패널**에 독립적으로 렌더링되는 결과물을 생성하는 기능

**지원 파일 형식**

| 타입 | 확장자 | 주요 용도 |
|------|--------|----------|
| Markdown | `.md` | 보고서, 문서, 강의 노트 |
| HTML | `.html` | 웹 페이지, 인터랙티브 UI |
| React | `.jsx` | 동적 컴포넌트, 대시보드 |
| SVG | `.svg` | 다이어그램, 일러스트 |
| Mermaid | `.mermaid` | 플로우차트, ER 다이어그램 |
| PDF | `.pdf` | 인쇄용 문서 |

**핵심 특징**
- **렌더링 패널 분리**: 코드 설명과 결과를 좌우로 동시에 확인
- **반복 수정(Iterative Editing)**: "표 컬럼 추가해줘", "색상 바꿔줘" 같은 후속 요청으로 점진적 개선 가능
- **다운로드 및 복사**: 생성된 결과물을 바로 활용 가능

---

#### JSX (React Component)

- React 기반 **인터랙티브 UI 컴포넌트** 렌더링
- 활용 예: 시뮬레이터, 계산기, 대시보드, 데이터 시각화(recharts)

**질의 예시**
```
다음 Claude 기능 설명을 jsx 형식으로 작성해줘
- 사용자 지정 / 프로젝트 / 아티팩트
```

---

#### SVG (Scalable Vector Graphics)

- XML 기반 벡터 그래픽으로 **대화창에 인라인 렌더링**
- 활용 예: 구조 다이어그램, 흐름도, 메모리 레이아웃 시각화, 개념 설명 일러스트

---

#### Mermaid 다이어그램

텍스트 기반으로 다이어그램을 생성하는 JavaScript 라이브러리

| 다이어그램 종류 | 활용 |
|----------------|------|
| `flowchart` | 알고리즘 흐름, 의사결정 과정 |
| `sequenceDiagram` | API 통신, 클라이언트-서버 상호작용 |
| `classDiagram` | OOP 설계, UML |
| `erDiagram` | 데이터베이스 스키마 |
| `stateDiagram-v2` | 오토마타, 상태 머신 |
| `gitGraph` | 브랜치 전략 설명 |

## 3. 스킬(Skill) 개요

### 3.1 스킬이란?

> **반복 가능한 작업 또는 워크플로를 위한 구조화된 지침을 미리 등록해 자동으로 적용하는 기능**

쉽게 말해, 특정 업무의 **표준 운영 절차(SOP)를 AI에 미리 주입**해두는 방법이다.

**특징 및 장점**

| 장점 | 설명 |
|------|------|
| 시간 절약 | 매번 좋은 프롬프트를 반복 작성할 필요 없음 |
| 토큰 비용 절감 | 필요할 때만 스킬 내용을 불러와서 사용 |
| 일관된 품질 | 표준화된 결과 생성 → 작업 품질의 일관성 확보 |

```
커스텀 스킬 = AI에게 전문가 핸드북 주입

SKILL.md 파일 하나로 → Claude가 상황에 맞게 자동 호출
프롬프트를 '기억'하는 구조가 아니라 ← 필요할 때 불러오는 시스템
```

---

### 3.2 스킬 작동 원리

```
1. 대화 시작 시
   → 스킬의 메타데이터(name, description)만 먼저 로드

2. 관련 요청이 들어오면
   → 전체 스킬 내용을 불러와 실행 (점진적 로딩)

3. 장점
   → 컨텍스트 과부하 방지
   → 활성화된 스킬만 선택적으로 사용해 응답 속도 향상
```

- 자연어로 워크플로를 정의하면 되므로 **코딩 없이도** 커스텀 스킬 생성 가능
- 이미 만들어진 스킬 라이브러리를 그대로 활용 가능

---

### 3.3 SKILL.md (핵심 파일)

모든 스킬은 `SKILL.md` 파일 하나로 시작한다. 두 부분으로 구성된다.

#### ① YAML 프론트매터 (항상 메모리에 상주)

```yaml
---
name: skill-name          # 스킬 식별자
description: |            # 자동 로드 판단 기준 + 스킬이 하는 일
  이 스킬은 ~할 때 사용한다.
  다음 상황에서 반드시 이 스킬을 사용하라: ...
---
```

- Claude는 이 부분만 보고 "이 요청에 이 스킬을 쓸지" 판단
- `description`에 **트리거 조건**을 명확히 작성하는 것이 핵심

#### ② 본문 Markdown (트리거 시 로드)

- 실제 작업 절차, 규칙, 템플릿
- **500줄 이하** 권장
- 상세 내용은 `references/` 폴더로 분리하고 포인터만 기재

---

### 3.4 스킬 폴더 구조

```
~/.claude/skills/
└── skill-name/           # 스킬 루트 폴더
    ├── SKILL.md           # 핵심 파일 (필수)
    ├── references/        # 상세 참조 문서 (선택)
    │   ├── guide.md
    │   └── templates.md
    ├── scripts/           # 자동화 스크립트, eval 측정 (선택)
    │   └── run_eval.py
    └── assets/            # 정적 파일, 템플릿, 폰트 등 (선택)
```

| 폴더 | 역할 |
|------|------|
| `references/` | SKILL.md에 넣기엔 큰 참조 문서. 필요할 때만 로드 |
| `scripts/` | 반복·자동화 작업용 실행 파일(Python 등), eval 스크립트 |
| `assets/` | 출력물 생성에 필요한 정적 파일 (템플릿, 폰트 등) |

## 4. 스킬 활용

### 4.1 스킬 활성화 방법

`왼쪽 사이드 메뉴 > 사용자 지정 > 스킬 > 내 스킬`

- 스킬 목록에서 원하는 스킬의 **토글 버튼**을 켜면 활성화
- 라이브러리에서 스킬을 추가하거나 직접 만든 스킬을 업로드 가능

---

### 4.2 스킬 직접 생성 절차

| 단계 | 내용 |
|------|------|
| **1. 반복 작업 선정** | 반복성·규칙성·일관성이 중요한 작업을 스킬 후보로 선정 |
| **2. 프롬프트 작성** | "이 작업용 스킬 만들어줘" + 목적 설명 |
| **3. Claude와 인터뷰** | 스킬 작성에 필요한 질의(객관식)에 응답 |
| **4. 스킬 생성 대기** | Claude가 SKILL.md를 자동 생성 |
| **5. 결과 확인** | description에 트리거 키워드 포함 여부, 본문 규칙·예시 확인 |
| **6. 수정 요청** | 필요하면 "다음 부분을 수정해서 다시 생성해줘" |
| **7. 테스트** | 자연스러운 문장으로 호출해 결과 확인 |
| **8. 반복 개선** | 만족스럽지 않으면 다시 수정 요청 |

---

### 4.3 스킬 활용 프롬프트 작성 팁

**나쁜 프롬프트 예시**
```
다음 정보를 바탕으로 "web-artifact-builder 스킬"을 활용해
인터랙티브한 강의 계획서 웹 페이지를 만들어줘
```
→ 스킬 이름은 내부 도구명이라 프롬프트에 명시할 필요 없음
→ 과목 내용이 없어서 Claude가 임의로 생성해야 하는 상황 발생

**개선된 프롬프트 예시**
```
"LLM과 AI Agent 입문" 과목의 인터랙티브 강의 계획서 웹 페이지를 만들어줘.

콘텐츠 요구사항
- 과목명: LLM과 AI Agent 입문
- 과목 개요(3~4문장)와 학습 목표(5가지 내외)를 상단에 배치
- 16주 커리큘럼: 주차별 학습주제, 세부 부주제 2~3개, 주요 과제 1개

UI/UX 요구사항
- 상단: 과목 개요 + 학습목표 섹션
- 중단: 전체 진도율 프로그레스 바 (현재 주차 클릭으로 설정)
- 하단: 16주 커리큘럼을 아코디언 방식으로 펼쳐보기
- 색상: 네이비(#1a2e5a)와 화이트 계열, 포인트 컬러 골드(#c9a84c)
- 반응형 레이아웃, 완료 주차는 체크 표시와 다른 색으로 구분
```

**좋은 프롬프트의 조건**
- 결과물의 **구체적인 구조와 UX** 요구사항 명시
- 필요한 **콘텐츠 내용을 직접 제공**하거나 생성 방향 안내
- 스킬 내부 도구명은 **프롬프트에 명시하지 않아도** 자동 트리거됨

```
Claude AI

claude.ai 주요 기능

├── 정의: SOP를 AI에 미리 주입해 반복 작업을 자동화
├── 핵심 파일: SKILL.md (YAML 프론트매터 + 본문 Markdown)

└── 생성: "이 작업용 스킬 만들어줘" → Claude와 인터뷰 → 테스트 → 등록
```
